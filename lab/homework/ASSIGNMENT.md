# 과제: 내 손으로 취약한 인프라 고치기

수업에서 배운 것을 **직접 처음부터 끝까지** 해보는 과제입니다.
수업 때와 다른 새 인프라(`homework/vulnerable/main.tf`)가 주어집니다.

## 상황
사내 웹서비스 인프라를 급하게 만들었더니 곳곳이 위험합니다. 직접 스캔하고 고쳐서 안전하게 만들어야 합니다.

## 할 일

### 1. 스캔
```bash
cd lab
./scan.sh tfsec   homework/vulnerable
./scan.sh checkov homework/vulnerable
```
- 취약 버전은 tfsec 기준 **CRITICAL 4 · HIGH 10** 정도가 뜹니다. (환경에 따라 조금 다를 수 있음)

### 2. 조치
`homework/vulnerable/main.tf` 를 복사해 직접 수정하세요. 목표는 하나입니다.
> **CRITICAL 과 HIGH 를 전부 0으로.**

힌트(수업에서 다 나온 것들):
- 공개 S3 → 비공개 + 암호화 + 퍼블릭 차단
- `0.0.0.0/0` 으로 열린 포트(SSH·MySQL) → 필요한 대역만
- 하드코딩 비밀번호 → 변수로
- 미암호화(RDS·EBS) → 암호화 켜기

### 3. 재스캔
```bash
./scan.sh tfsec homework/<내가_고친_폴더>
```
CRITICAL·HIGH 가 0이면 성공입니다. (MEDIUM·LOW 는 남아도 괜찮습니다: '있으면 더 좋은' 권고입니다.)

## 제출물

아래 항목을 **PDF 한 개 파일**로 묶어 제출합니다.

1. **고친 `main.tf`** (전체 코드)
2. **스캔 결과 캡처**: 고치기 전(critical/high 있음) vs 고친 후(critical/high 0)
3. **수정 이유**: 무엇을 왜 고쳤는지 항목별로 한 줄씩
4. **한 줄 회고**: 가장 위험하다고 느낀 항목 하나

> 평가는 실제로 동작하는 결과물(고친 코드 · 재스캔 수치 · 정확한 수정 이유)을 중심으로 합니다. 서술형 회고는 한 줄이면 충분합니다.
